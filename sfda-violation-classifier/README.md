# دليل تصنيف المخالفات

برنامج استرشادي للمفتش يرشح أقرب تصنيفات مخالفة محتملة بناءً على وصف الحالة، اعتماداً على جداول تصنيف المخالفات الرسمية الصادرة عن الهيئة العامة للغذاء والدواء في يوليو 2026.

## التشغيل والاختبار

```bash
npm install
npm test
npm start
```

## استيراد بيانات جداول PDF

لا يقرأ التطبيق ملفات PDF أثناء البحث. بعد استخراج أي إصدار جديد إلى ملف مرحلي خارج مجلد `public`، مثل `imports/violations.raw.json`، يُعتمد بالأمر:

```bash
npm run approve-pdf-data -- --input=imports/violations.raw.json
```

يمر الاعتماد بالمراحل الإلزامية التالية:

1. تطبيع النص العربي وإصلاح أنماط PDF/OCR المعروفة، مثل الكلمات المقلوبة أو الملتصقة أو المنقسمة.
2. تدقيق جميع الصفوف وإيقاف العملية عند بقاء أي نمط استخراج مشبوه، أو نقص أي حقل أو قطاع أو مرجع.
3. إنشاء `violations.json` معتمد، وتوليد بيان `violations.manifest.json` يتضمن العدد وتوزيع القطاعات وبصمة SHA-256.
4. قراءة محرك البحث للنسخة المعتمدة فقط؛ ولا تُنشر ملفات الاستخراج المرحلية.

ينفذ `npm run build` فحص القفل والبصمة قبل إنشاء نسخة النشر. أي تعديل غير معتمد أو خطأ OCR متبقٍ يوقف البناء، ولا يعيد البناء كتابة البيانات تلقائيًا.

## النشر على Vercel

اربط هذا المستودع بمشروع Vercel جديد. إعدادات البناء مضمنة في `vercel.json`، وسيتم تنفيذ `npm run build` ونشر مجلد `dist` تلقائياً.

البحث يعمل محلياً داخل المتصفح ولا يرسل وصف الحالة إلى خدمة خارجية.
